# 微博互关自动助手（Android / 小米）

版本 4.1 将名单扫描、批量私信和批量评论彻底拆分，并且只处理用户勾选的好友。

## 扫描并选择打卡对象

1. 最直接的方法：停在官方微博互相关注列表，点击浮窗中间的“扫描名单”，无需离开微博。
2. 也可以切到助手点击“扫描/刷新互关名单”；助手会退到后台，直接露出刚才的微博页面，再点浮窗“开始扫描”。
3. 扫描会先自动回到列表顶部，再用带重叠的小步滑动完整扫描两遍，只保存去重后的昵称，不会读取或保存 ID，也不会发送任何内容。
4. 扫描完成后回到 App，在名单中勾选需要打卡的人并点击“保存选择”。
5. 再回互关列表启动正式任务；未勾选好友会直接跳过。

扫描只保存微博页面向无障碍服务暴露的昵称。两轮扫描结果取并集并按昵称去重，减少大幅滑动造成的漏人和人数波动。

扫描只支持官方微博 App 包 `com.sina.weibo`，不支持网页版。若官方 App 未安装或系统暂时无法识别，助手只提示错误，不再自动跳转浏览器。

## 速度与后台限制

- “每步等待时间”现在可设为 1～15 秒。建议先用 2～3 秒，稳定后再尝试 1 秒。
- 输入文字、发送按钮和评论输入框仍会通过页面状态确认，不能安全省略的等待不会被强行跳过。
- 无障碍自动化不能真正后台运行：微博必须保持前台、屏幕保持解锁，否则 Android 不提供可操作页面控件。后台私有接口或 Root 方案不在本工程中使用。

## 独立的私信与评论任务

- 浮窗空闲时显示三个入口：**发私信、发评论、扫描名单**。
- “发私信”只执行私信流程，成功后返回互关列表，不会进入评论流程。
- “发评论”只进入主页查找微博并评论，不会打开私信。
- App 内也提供“只执行批量私信”和“只执行批量评论”按钮。
- 运行中三个浮窗按钮会变为：**暂停/继续、跳过此人、停止**。

浮窗提供 **暂停/继续、跳过此人、停止**。整个发送流程不要求逐次人工确认。

## 去重规则

- 去重只在当天有效，记录形式为 `日期 + 好友昵称`；
- “已私信”和“已评论”使用两个互不相关的记录库；
- 私信成功只写入已私信库，评论成功只写入已评论库；
- 同一天暂停、重启或再次运行不会重复执行对应的已成功任务；
- 第二天自动重新开始全部互关好友；
- “跳过此人”只在本轮跳过，不会写入成功记录；
- App 首页可以手动清空今天的已处理记录。

私信点击发送后只检查“内容相同、请隔十分钟、操作频繁、稍后再试、发送失败”等明确警告；没有警告就记为已私信，不比较消息内容和时间。评论仍会比较提交前后的新增内容，并结合当前或上一分钟的时间验证成功。检测到限制警告时会自动暂停，不写入成功库，并把该对象记为当天限流跳过；继续或重新启动时从后续对象开始。

## 四项设置

1. **自动私信内容**：进入每位好友私信页后发送的文字，默认 `续个火花✨`。
2. **主页评论内容**：私信后在好友主页微博下发送的评论，默认 `踩踩宝贝`。
3. **每步等待时间**：范围 1～15 秒，建议先用 2～3 秒，稳定后可尝试 1 秒。
4. **本轮最多处理人数**：填 `0` 表示处理到列表底部；也可以填 1～500 设置单轮上限。

## 手机上使用

1. 安装 APK，打开 App 设置内容和速度并保存。
2. 开启“微博互关自动助手”无障碍服务。
3. 首次按“扫描并选择打卡对象”的步骤建立目标名单。
4. 在微博进入“互相关注”好友列表，并停留在列表顶部。
5. 点击右上角浮窗“发私信”或“发评论”。
6. 需要干预时点击“暂停”或“停止”。

小米/澎湃 OS 请允许自启动，将省电策略设为“不限制”，并把应用锁定在最近任务中。

## GitHub Actions 构建 APK

工程包含 `.github/workflows/build-apk.yml`，推送代码后会自动构建：

```powershell
git add .
git commit -m "拆分名单扫描私信和评论任务"
git push
```

在 GitHub `Actions` 打开最新成功运行，从 `Artifacts` 下载 `WeiboSparkAssistant-debug-apk`，解压得到 `app-debug.apk`。

## 页面适配说明

微博没有为此流程提供公开稳定接口，程序依赖“互相关注、私信、发送、评论”等可访问文本及控件位置。如果微博版本更新后某一步无法确定，程序会暂停而不是盲目点击；App 首页会保存最近一次异常的阶段和源码行号。
