# 微博火花助手（Android / 小米）

这是一个独立 Android 工程，通过无障碍服务在微博前台完成续火花流程。

新版流程不依赖“互相关注”列表，而是从微博底部 **消息** 页面识别带有“火花”图标或无障碍描述的会话：

1. 打开一个当天未处理的火花会话；
2. 发送设置好的续火花消息；
3. 从聊天页进入对方主页；
4. 打开第一条可见微博的评论入口并发送主页留言；
5. 自动返回消息列表并继续下一位。

## 第一次学习与后续自动执行

- 第一次使用时，私信和评论各确认一次，用于核对页面和记录微博当前版本的输入框、发送按钮、主页入口与评论入口。
- 第一个联系人完成后，应用保存为“已学习”。之后的联系人会自动发送、自动评论、自动返回消息列表，不再逐人确认。
- 微博升级导致页面控件变化时，可在应用中点击“重新学习微博操作步骤”。
- 已处理记录按日期保存：同一天避免重复；当天后来新增的火花联系人，再启动一次仍可处理；第二天会重新识别。

> 火花图标必须向 Android 暴露“火花”“连续聊”“🔥”等文字或无障碍描述才能被可靠识别。如果微博把它做成完全无描述的图片，应用会暂停，不会把普通会话当成火花会话误发。此时可先手动打开一个火花聊天，再点浮窗继续完成首次学习。

## 四项设置是什么意思

1. **续火花消息**：发到带火花标识聊天中的消息，默认 `续个火花✨`。
2. **主页留言**：发完聊天消息后，到对方主页微博下发送的评论，默认 `踩踩宝贝`。
3. **每步等待时间**：给微博打开聊天、主页和评论框留出的加载时间；小米建议 4～8 秒。
4. **本轮最多处理人数**：一次启动最多处理多少人，到达后自动停止；范围 1～50。

## GitHub Actions 云端构建 APK

工程包含 `.github/workflows/build-apk.yml`，本机无需安装 Android SDK 或 Gradle。

1. 提交修改并推送到 GitHub：

   ```powershell
   git add .
   git commit -m "改为从消息页自动续火花"
   git push
   ```

2. 打开仓库的 `Actions` 页面，选择 `Build Weibo Spark APK`。
3. 可以等待推送自动触发，也可以点击 `Run workflow` 手动触发。
4. 构建成功后点击运行编号（例如 `Build Weibo Spark APK #2`）回到运行汇总页。
5. 在 `Artifacts` 中下载 `WeiboSparkAssistant-debug-apk`，解压得到 `app-debug.apk`。

当前版本为 1.3。首次从先前的 1.0 调试包升级时，如果手机提示签名不一致，请先卸载旧版再安装一次；工作流现已缓存固定调试签名，之后的新构建通常可以直接覆盖安装。

## 手机上使用

1. 安装 APK，设置四项内容并保存。
2. 开启“微博火花助手”无障碍服务。
3. 打开微博底部“消息”页，让火花会话出现在屏幕上。
4. 第一次点击浮窗“开始学习”，按提示完成两次首次确认。
5. 学习完成后，以后进入消息页点击“开始续火花”即可自动处理。

小米/澎湃 OS 若频繁停止服务：在应用详情中允许自启动，将省电策略设为“不限制”，并把应用锁定在最近任务中。

如果无障碍页提示“该服务出现故障”，请重新打开助手首页。1.2 及以上版本会把最近一次异常的阶段、类型和源码行号显示在顶部状态卡中；长按即可复制。自动流程遇到异常会安全暂停，不再让整个无障碍服务直接崩溃。

1.3 版已适配当前微博评论提交按钮显示为“评论”的界面，并优先选择靠近评论输入框、位于屏幕右侧的按钮，避免误点微博卡片中的普通评论入口。

## 保护与限制

- 只处理明确包含火花文字或无障碍描述的会话，不把普通聊天当作目标；
- 每步间隔限制为 3～15 秒；
- 单轮人数限制为 1～50；
- 当天按联系人去重；
- 任何时候可通过浮窗暂停或停止；
- 只读取微博包 `com.sina.weibo` 的前台页面，不联网、不读取账号密码。

微博没有为此流程提供公开稳定接口，页面文字、控件 ID 或布局变化都可能让某一步暂停。应用会优先复用首次学习的控件 ID，失效时回退到文字和页面位置识别；仍无法确定时会停止等待处理，而不是盲目点击。
