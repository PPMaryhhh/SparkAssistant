# 微博互关自动助手（Android / 小米）

版本 2.3 从微博 **互相关注列表** 全自动逐人处理。页面可以通过顶部标题或多个好友行右侧的“互相关注”标识识别；好友主页上的单个关系按钮不会再被误判成列表。

## 自动操作流程

1. 在互相关注列表选择当天未处理的第一位好友；
2. 进入好友主页并点击“私信”；
3. 点击并聚焦输入框，填写私信后读取页面确认文字真正显示；直接写入不生效时自动改用剪贴板粘贴；
4. 关闭输入状态，再返回好友主页；
5. 点击主页任意一条可评论微博的评论入口，进入微博后再点击屏幕底部的消息/评论图标；图标没有文字描述时，按底部工具栏结构识别，并以底部中央坐标点击兜底；
6. 输入框出现后填写评论内容，等待并点击当前微博版本的“评论”提交按钮；
7. 评论完成后至少执行一次返回，再持续返回到互相关注列表并继续下一位；
8. 当前屏幕好友全部处理后自动下滑，直到列表底部。

浮窗提供 **暂停/继续、跳过此人、停止**。整个发送流程不要求逐次人工确认。

## 去重规则

- 去重只在当天有效，记录形式为 `日期 + 好友昵称`；
- 同一天暂停、重启或再次运行都不会重复处理已经成功私信和评论的好友；
- 只有私信和评论都完成后才写入当天完成记录；
- 第二天自动重新开始全部互关好友；
- “跳过此人”只在本轮跳过，不会写入成功记录；
- App 首页可以手动清空今天的已处理记录。

## 四项设置

1. **自动私信内容**：进入每位好友私信页后发送的文字，默认 `续个火花✨`。
2. **主页评论内容**：私信后在好友主页微博下发送的评论，默认 `踩踩宝贝`。
3. **每步等待时间**：给微博页面加载和键盘收起留出的时间，小米建议 4～8 秒。
4. **本轮最多处理人数**：填 `0` 表示处理到列表底部；也可以填 1～500 设置单轮上限。

## 手机上使用

1. 安装 APK，打开 App 设置四项内容并保存。
2. 开启“微博互关自动助手”无障碍服务。
3. 在微博进入“互相关注”好友列表，并停留在列表顶部。
4. 点击右上角浮窗“开始”。
5. 需要干预时点击“暂停”或“停止”。

小米/澎湃 OS 请允许自启动，将省电策略设为“不限制”，并把应用锁定在最近任务中。

## GitHub Actions 构建 APK

工程包含 `.github/workflows/build-apk.yml`，推送代码后会自动构建：

```powershell
git add .
git commit -m "改为互关列表全自动处理"
git push
```

在 GitHub `Actions` 打开最新成功运行，从 `Artifacts` 下载 `WeiboSparkAssistant-debug-apk`，解压得到 `app-debug.apk`。

## 页面适配说明

微博没有为此流程提供公开稳定接口，程序依赖“互相关注、私信、发送、评论”等可访问文本及控件位置。如果微博版本更新后某一步无法确定，程序会暂停而不是盲目点击；App 首页会保存最近一次异常的阶段和源码行号。
